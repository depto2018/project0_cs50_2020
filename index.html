What is AWS?

AWS or Amazon Web Services is a cloud computing platform which provides a bunch of computational resources on the cloud offering a pay as you go model meaning that you pay only for the resources you use on a certain period of time. The benefits of using the power of a cloud computing solution like this one is the ability to have your infrastructure deployed a lot faster than the time you will take to deploy an on-premise solution where you need to start from racking your devices on a datacenter, cabling, and considering all the physical security needs you need to take into account for this. Apart from that, you will also need to take care of the OS installation and applications deployment which will also require big efforts. Cloud computing eases the deployment of all resources for you.

Big competitors of AWS nowadays are Google Cloud and Microsoft Azure. However, AWS is the actual lead on the market.

AWS infrastructure

The AWS infrastructure is composed by Regions which are physical geographic locations around the world where datacenters are deployed. Within the regions we find Avaliability Zones (AZ) which are groups of interconnected datacenters. Likewise, AZs within a region are connected through high speed lines for Avaliability and Reliability porpuses. There are also other infrastructure concepts like AWS Local Zones, AWS wavelength and AWS Outsposts which are meant to provide very low latency scenarios for development and testing purposes. How do you decide which Region to choose?

Important factors you need to take into account for choosing the correct region are the following:

  • Latency: Closer to end users.
  • Cost: Every region may have different costs.
  • Compliance: Legal restrictions according to location policies.
  • Service availability: New services may run on specific regions only.
