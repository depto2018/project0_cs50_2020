AWS S3 (Simple Storage Service)

What is AWS S3?

AWS S3 is an object (file) level storage service which can be used to save general purpose data on the cloud and access it from anywhere. It can be seen as a NAS (Network Attached Storage) in an on-premise network. General features of AWS S3 are the following:

  • Data is distributed across at least three AZs with the exception of One Zone-IA class S3 storage where the data is kept in one AZ only, which makes it a cheaper solution.
  • It supports encryption.
  • Provides automatic data classification. For example if data is not manipulated in any way during a certain period of time, it can be automatically moved to AWS Glacier for long term storage.
  • Useful for big data analytics.

Some AWS S3 concepts...

Objects

Objects in S3 have eventual consistency, which means that anytime an object is modified it will get replicated in all AZs but it might take some time as a difference with AWS EBS where objects are consistent.

Buckets

Buckets are basically containers to store the objects.

REST

S3 is based on REST (Representational State Transfer) which is a system interface that maps HTTP methods to CRUD operations:

  • Create → Put or Post operation.
  • Read → Get operation.
  • Update → Post or Put operation.
  • Delete → Delete operation.

Note: It is more common to use Put operations for create actions and Post for update actions, but they can be interchanged.

How to put data into S3?

There are different ways of getting data into S3 as follows:

  • Uploading the files manually to an S3 bucket from your local desktop like the way you do with a storage cloud service like google drive or dropbox.
  • Files can be uploaded through an API (Application Programming Interface).
  • Amazon Direct Connect can be used to upload files through a VPN.
  • An Storage Gateway can also be used to synchronize local data into AWS S3.
  • Through Kinesis Firehose?
  • Transfer acceleration and optimized data access can be done through AWS cloud front.

S3 storage classes

There are different types of storage classes which will offer you more or less benefits with different cost chargings. Storage classes can be categorized as follows:

  1. General Purpose
    • Standard: This is to store fequently accessed data with low latency and high throughput performance. Ideal for web applications, gaming, big data, etc.
    • Intelligent-tiering: This is for unkonwn or changing access. It automatically moves objects between two access tiers (Frequent and Infrequent tiers) based on changing access patterns.
  2. Infrequent access (IA)
    • Standard IA: This is for data that is accessed less frequently, but require rapid access if needed. Ideal solution for backups and disaster recovery data.
    • One Zone-IA: This class provides same features as standard-IA, but data is stored in one AZ only. In any of the other cases data is stored on a minimum of 3 AZs. Ideal for secondary backups or easily re-creatable data.
  3. Archive
    • Glacier: Low cost class for data archiving. It provides different retrieval options ranging from minutes to hours based on customer needs.
    • Glacier deep archive: Lowest cost storage class. It is designed for long-term retention data that will be retained for 7-10 years. Data can be retrieved within 12 hours.

Data can be moved from standard to either IA or Glacier based on time policies known as S3 Object Lifecycle Management. For further detailed information, you can go to the AWS website: AWS S3.

Some S3 facts...

  • ⇒ A single object can be up to 5 TB in size.
  • ⇒ Bucket names must be globally unique.
  • ⇒ S3 classes can be configured at the object level and a single bucket may have objects stored using different classes.
  • ⇒ Objects versioning can be suspended but not Disabled.
  • ⇒ Objects inherit permissions from the bucket's but they can be overwritten at the object level.
  • ⇒ Tags are used to search, organize and manage object access. An object can have up to 10 tags.
  • ⇒ When you delete an object, it is only marked for deletion at first time so you can recover it later.
