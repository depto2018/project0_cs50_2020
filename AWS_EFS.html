AWS EFS (Elastic File System)

IOPS: Input/output Operations per Second

General Purpose SSD (gp2) is the default volume for EC2 instances.

To maximize the performance of any EBS volume, AWS recommends to use EBS-Optimized EC2 instances.

Some AWS EBS facts...

  • The Amazon data lifecycle manager can be used to automate the process of taking EBS snapshots according to user defined policies.
  • EBS volumes are elastic which means, anytime the user may increase capacity, tune performance and change the type of any volume with no downtime or impact.
  • EBS volumes and snpashots can be encrypted with general or user created keys.
  • Amazon EBS Multi-Attach allows an EBS io1 volume to be attached concurrently to up to 16 Nitro-based EC2 insatnces within the same AZ.
  • EC2 instances and EBS volumes need to be on the same AZ.
